# Capítulo 6: Bibliografía y Referencias

A continuación se listan las principales fuentes bibliográficas, documentación técnica y referencias a estándares oficiales consultadas para el diseño, desarrollo e implementación del ecosistema **BioField**:

## 6.1.- REFERENCIAS DE ARQUITECTURA Y METODOLOGÍA

* **Evans, E. (2003)**. *Domain-Driven Design: Tackling Complexity in the Heart of Software*. Addison-Wesley Professional. (Referencia fundamental para el modelado de capas y entidades de dominio del backend C#).
* **Martin, R. C. (2017)**. *Clean Architecture: A Craftsman's Guide to Software Structure and Design*. Prentice Hall. (Principios de desacoplamiento de dependencias y aislamiento del núcleo comercial).
* **Beck, K. (2001)**. *Manifiesto por el Desarrollo Ágil de Software*. (Fundamento de la organización del proyecto en hitos iterativos Scrum).

## 6.2.- DOCUMENTACIÓN DE BACKEND Y BASES DE DATOS

* **Microsoft Corporation**. *Documentación oficial de .NET 9.0 y ASP.NET Core*. Recuperado de: [https://learn.microsoft.com/dotnet/](https://learn.microsoft.com/dotnet/) (Referencia de configuración de inyección de dependencias, controladores de API Minimal, enrutado y middleware).
* **Entity Framework Core**. *EF Core Documentation*. Recuperado de: [https://learn.microsoft.com/ef/core/](https://learn.microsoft.com/ef/core/) (Manejo de migraciones, mapeo de enums, relaciones N a N y tracking de entidades).
* **PostgreSQL Global Development Group**. *PostgreSQL 16 Documentation*. Recuperado de: [https://www.postgresql.org/docs/16/](https://www.postgresql.org/docs/16/) (Administración, índices de claves y optimización relacional).
* **MinIO Inc**. *MinIO Object Storage Documentation*. Recuperado de: [https://min.io/docs](https://min.io/docs) (Integración y configuración del almacenamiento compatible con Amazon S3 API).

## 6.3.- DOCUMENTACIÓN DE CLIENTES (MÓVIL Y WEB)

* **Google Flutter Team**. *Flutter & Dart Documentation*. Recuperado de: [https://docs.flutter.dev/](https://docs.flutter.dev/) (Ciclo de vida de widgets de Material Design, consumo de sensores de geolocalización y renderizado nativo).
* **Drift Library**. *Drift (Moor) SQLite documentation for Flutter*. Recuperado de: [https://drift.simonbinder.eu/](https://drift.simonbinder.eu/) (Modelado de tablas reactivas SQLite, generación de código estático y soporte offline).
* **Vite Core Team**. *Vite Guide & Configuration*. Recuperado de: [https://vite.dev/](https://vite.dev/) (Compilación e bundling de la aplicación React 19).
* **MapLibre GL Team**. *MapLibre GL JS Documentation*. Recuperado de: [https://maplibre.org/maplibre-gl-js/docs/](https://maplibre.org/maplibre-gl-js/docs/) (Cámara 3D tridimensional acelerada por WebGL y renderizado de teselas raster y vectoriales).
* **Turf.js Contributors**. *Turf.js Geospatial Analysis library*. Recuperado de: [https://turfjs.org/](https://turfjs.org/) (Funciones matemáticas de cálculo de bearing, distancias geodésicas e interpolación de líneas).

## 6.4.- SERVICIOS Y ESTÁNDARES

* **Google Cloud Identity**. *Google Sign-In OAuth2 Integration*. Recuperado de: [https://developers.google.com/identity](https://developers.google.com/identity) (Flujo de autenticación segura).
* **Google Generative AI**. *Gemini API Reference*. Recuperado de: [https://ai.google.dev/gemini-api/docs](https://ai.google.dev/gemini-api/docs) (Integración del modelo inteligente de análisis ecológico contextual).
* **Docker Inc**. *Docker Compose Specification*. Recuperado de: [https://docs.docker.com/compose/](https://docs.docker.com/compose/) (Orquestación multinivel del ecosistema en desarrollo y producción).
